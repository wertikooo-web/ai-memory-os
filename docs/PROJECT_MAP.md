# AI Memory OS: от нуля до полной версии

## Главная идея

AI Memory OS помогает человеку не держать всю жизнь в голове.

Он принимает хаос: мысли, дела, голосовые, ссылки, документы, обещания. Потом раскладывает это по контекстам и каждый день показывает, на чем реально сфокусироваться.

Короткая формула проекта:

```text
AI Memory OS принимает хаос,
превращает его в контексты,
следит за открытыми циклами
и каждое утро показывает человеку,
что действительно требует внимания сегодня.
```

## 0. Фундамент

```text
Telegram Bot
  -> Input Normalization
  -> Memory Core
  -> Supabase/Postgres
  -> Open Cycles / Contexts / Memory Items
```

Что уже есть:

- бот работает;
- база подключена;
- сообщения сохраняются;
- голосовые сообщения распознаются;
- OpenCycles создаются;
- OpenCycles можно закрывать естественной фразой;
- удаление работает через безопасное подтверждение;
- `/today` строит первый Morning Focus.

## 1. Вход данных

Пользователь просто пишет или говорит:

```text
купить помидоры
ответить по Японии
запиши в дела по дому: повесить занавески
идея для AI Memory OS: утренний фокус
```

Система должна принять все.

Главное правило:

```text
сначала сохранить
потом разобрать
```

Запись не должна теряться из-за ошибки LLM, транскрибации, классификации или выбора контекста.

## 2. Input Normalization

Любой источник информации сначала нормализуется, а потом вся система работает только с единым форматом данных.

Источники могут быть разными:

- текст Telegram;
- голосовое сообщение;
- фотография;
- PDF;
- документ;
- ссылка;
- email в будущем;
- запись с медальона в будущем;
- данные из других интеграций.

Основной pipeline:

```text
Input
  -> Input Normalization
  -> Normalized Text
  -> LLM Classification / Natural Intent
  -> Context
  -> Open Cycle / Memory Item
  -> Morning Focus
```

Правила:

- если вход уже текстовый, вернуть текст;
- если голос, выполнить транскрибацию и вернуть текст;
- если изображение, выполнить OCR или Vision-анализ и вернуть текстовое описание;
- если PDF или документ, извлечь текст;
- если ссылка, получить заголовок, основной текст и краткое описание.

После Input Normalization следующие модули работают только с текстом и метаданными. Memory Core не должен знать, пришла запись из Telegram, голосового сообщения, PDF, фотографии или будущего устройства.

## 3. Контексты

Не все должно падать в одну кучу.

Примеры контекстов:

- Inbox;
- Япония;
- AI Memory OS;
- VoiceBridge;
- Дела по дому;
- Покупки;
- Весь бред;
- Семья;
- Здоровье.

Пользователь может сам сказать:

```text
запиши в Японию: написать партнеру
```

Если не сказал, бот кладет запись в Inbox или позже предлагает контекст.

Контекст - это не просто папка. Это жизненная область, проект, поездка, тема, человек, встреча или рабочий фокус.

## 4. Open Cycles

Главная рабочая сущность.

`OpenCycle` = незавершенная вещь, которую человек держит в голове.

Это может быть:

- дело;
- обещание;
- покупка;
- идея;
- вопрос;
- ожидаемый ответ;
- проектный хвост;
- бытовая мелочь.

Примеры:

- купить помидоры;
- ответить Масаки;
- подкачать колесо;
- дописать архитектуру;
- позвонить папе.

Смысл Open Cycles: система берет на себя то, что человек обычно держит в голове.

## 5. LLM-слой

LLM не должен быть “магическим чатом”.

Его первая работа: понять входящее сообщение.

Он определяет:

- тип: задача / идея / покупка / обещание / заметка;
- контекст: Япония / дом / AI Memory OS;
- срочность;
- важность;
- энергию;
- примерное время;
- дату, если есть;
- почему он так решил.

LLM используется как сервис понимания входа, а не как самостоятельный собеседник.

## 6. Morning Focus

Главная функция MVP.

Каждое утро бот показывает:

```text
Главный фокус дня
Сделать сегодня
Если останется время
Можно не держать в голове
```

Смысл:

- у пользователя может быть 125 открытых циклов;
- сегодня он видит только 3-7 важных вещей;
- остальное система держит сама.

Morning Focus - это не просто список задач. Это способ снизить mental load.

## 7. Proactive Notifications

Бот может писать сам.

Но редко.

Допустимые причины:

- утренний фокус;
- важное напоминание;
- зависший открытый цикл.

Правило:

```text
бот не должен раздражать
каждое сообщение должно иметь причину
```

Proactive Notifications должны помогать человеку чувствовать контроль, а не давление.

## 8. Telegram Mini App

Бот остается быстрым входом.

Mini App становится экраном.

```text
Bot = быстро кинуть мысль
Mini App = увидеть день и порядок
```

Главные экраны:

- Today View;
- Open Cycles;
- Contexts;
- Mental Load;
- Settings.

Mini App не заменяет бота. Он показывает структуру, порядок и фокус поверх того же Memory Core.

## 9. Voice

Голосовой сценарий:

```text
голос
  -> Input Normalization
  -> транскрибация
  -> Normalized Text
  -> LLM-классификация / Natural Intent
  -> контекст
  -> Open Cycle / Memory Item
  -> Morning Focus
```

Голос нужен, чтобы быстро выгрузить мысль из головы.

Пользователь не должен думать, как правильно оформить запись. Он просто говорит.

## 10. Search / Recall

Позже пользователь спрашивает:

```text
что у нас было по Японии?
что я думал про медальон?
какие хвосты по дому?
почему мы отложили VoiceBridge?
```

Система отвечает не списком файлов, а структурированной выдержкой.

Recall должен восстанавливать контекст, а не просто искать слова.

## Ориентир для AI-агентов

Этот документ является короткой картой проекта для Codex, Claude Code и будущих AI-агентов.

Перед крупными изменениями нужно сверяться с ним и проверять:

- не превращаем ли мы продукт в обычный заметочник;
- не дублируем ли бизнес-логику между интерфейсами;
- не показываем ли пользователю слишком много хаоса;
- помогает ли изменение снижать mental load;
- сохраняется ли правило “сначала сохранить, потом разобрать”;
- проходит ли любой новый источник через Input Normalization.
