# ROADMAP

## Этап 1. Фундамент проекта

Создать документацию и структуру репозитория:

- README;
- PROJECT;
- VISION;
- ARCHITECTURE;
- ROADMAP;
- AGENTS;
- DECISIONS;
- GLOSSARY;
- prompts.

Цель: чтобы Codex, Claude Code и человек одинаково понимали проект.

## Этап 2. Минимальный Telegram MVP

Создать Telegram bot.

Минимальные команды:

- `/start`;
- `/events`;
- `/last`;
- `/delete_last`.

Любое текстовое сообщение сохраняется как `MemoryItem` в `Inbox`.

## Этап 3. Supabase/Postgres

Подключить Supabase как PostgreSQL-базу через `DATABASE_URL`.

Создать таблицы:

- `User`;
- `LifeEvent`;
- `MemoryItem`;
- `Asset`.

## Этап 4. Railway Deploy

Статус: выполнено.

MVP задеплоен на Railway и подтвержден в живом Telegram-боте.

Текущая схема:

```text
Railway -> Node.js/grammY long polling -> Supabase/Postgres
```

Цель этапа достигнута: бот работает постоянно, без запуска с локального компьютера.

Важное ограничение: при long polling должен работать только один экземпляр бота. Локальный запуск и Railway одновременно могут вызвать `409 Conflict` от Telegram.

## Этап 5. Голосовые сообщения

На первом шаге бот принимает голос.

Позже подключается OpenAI для транскрибации.

Результат транскрибации становится `MemoryItem`.

## Этап 6. Управление LifeEvent

Добавить базовые действия:

- создать событие;
- переименовать событие;
- переместить последнюю запись;
- показать записи события.

## Этап 7. Search/Recall

Добавить простой поиск и восстановление контекста:

- поиск по тексту;
- поиск по событию;
- показать последние записи;
- ответить на вопрос по сохраненным записям.

## Этап 8. Ссылки, фото и документы

Добавить прием:

- ссылок;
- фото;
- документов.

Сначала сохранять как `Asset`, затем добавлять анализ.

## Этап 9. OpenAI-анализ

Подключить OpenAI для:

- транскрибации;

- классификации;

## Этап 10. Новые интерфейсы

- Web;

Новые интерфейсы не должны обходить Memory Core.
