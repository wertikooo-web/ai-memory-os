# GLOSSARY

## AI Memory OS

Персональная внешняя память человека.

Система, которая принимает фрагменты жизни, связывает их с контекстом и помогает потом восстановить смысл.

## Personal Memory Engine

Ядро продукта, которое отвечает за сохранение, организацию и поиск памяти.

## Memory Core

Внутренний слой логики памяти.

Он не должен зависеть от Telegram, Web, Mobile или других интерфейсов.

## LifeEvent

Главная сущность продукта.

Событие жизни, проект, встреча, поездка, тема или рабочий контекст.

Примеры:

- проект;
- встреча;
- поездка;
- человек;
- идея;
- решение;
- период жизни.

## MemoryItem

Отдельная запись внутри `LifeEvent`.

Примеры:

- текст;
- голос;
- фото;
- документ;
- ссылка;
- идея;
- задача;
- заметка.

## Asset

Оригинальный файл или объект.

Примеры:

- аудио;
- фото;
- документ;
- ссылка;
- вложение из Telegram.

## Inbox

Событие по умолчанию.

Если система не понимает контекст записи, она сохраняет запись в `Inbox`.

## Interface

Точка входа пользователя в память.

Примеры:

- Telegram;
- Web;
- Mobile;
- Voice Pendant;
- Smart Watch;
- API.

## Search

Поиск по сохраненным записям.

На первом этапе может быть простым текстовым поиском.

## Recall

Восстановление контекста.

Это не просто поиск строки, а ответ на вопрос пользователя на основе сохраненной памяти.
## OpenCycle

Структурированное понимание входящего сообщения.

`OpenCycle` не заменяет `MemoryItem`. `MemoryItem` хранит исходную запись, а `OpenCycle` хранит интерпретацию: что это было, к какой области относится, насколько срочно и важно, сколько энергии требует и есть ли срок.

Примеры типов:

- `TASK`;
- `THOUGHT`;
- `PURCHASE`;
- `IDEA`;
- `PROMISE`;
- `NOTE`;
- `OTHER`.
