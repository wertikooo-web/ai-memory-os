# PROJECT

## Что мы строим

AI Memory OS — персональная внешняя память человека.

Система принимает фрагменты жизни и работы, сохраняет их в едином ядре памяти, связывает с контекстом и помогает позже найти нужное.

Пользователь должен ощущать продукт не как базу данных и не как бота, а как место, куда можно быстро отправить жизнь:

- мысль;
- голос;
- фото;
- документ;
- ссылку;
- идею;
- задачу;
- обещание;
- заметку после встречи;
- материал по проекту.

## Главный принцип

Пользователь никогда не должен думать, куда сохранить информацию.

Сначала сохранить.

Потом понять.

Потом связать.

Организация памяти никогда не должна мешать фиксации мысли.

## Миссия

Помочь человеку не терять контекст собственной жизни.

AI Memory OS должен помнить то, что человек обычно забывает:

- что обсуждалось;
- почему было принято решение;
- кому что обещали;
- где лежит важный материал;
- как развивалась идея;
- что происходило вокруг проекта, встречи или поездки.

## Главная сущность

Главная сущность продукта — `LifeEvent`.

`LifeEvent` — это событие жизни, проект, встреча, поездка, тема или рабочий контекст.

Примеры:

- `VoiceBridge`;
- `AI Memory OS`;
- `Поездка в Японию`;
- `Встреча с инвестором`;
- `Здоровье июль`;
- `Поиск квартиры`;
- `Идеи для медальона`.

## Внутренние сущности

`MemoryItem` — отдельная запись внутри `LifeEvent`.

Это может быть:

- текст;
- голос;
- фото;
- документ;
- ссылка;
- идея;
- задача;
- решение;
- заметка.

`Asset` — оригинальный файл или объект:

- аудио;
- фото;
- документ;
- ссылка;
- вложение из Telegram.

## Контекст важнее хранения

Мы не просто храним память.

Мы сохраняем контекст жизни.

Голос, фото, PDF и ссылка по отдельности мало что значат. Но если система понимает, что все это относится к одной встрече, поездке или проекту, она начинает работать как внешняя память.

Именно контекст отличает AI Memory OS от AI-диктофона, заметочника и файлового архива.

## Что не является целью

На первом этапе AI Memory OS не должен быть:

- CRM;
- календарем;
- файловым хранилищем;
- полноценным таск-менеджером;
- web-приложением;
- mobile-приложением;
- сложной AI-платформой.

Эти функции могут появиться позже, но они не должны ломать главную идею: внешняя память вокруг жизненных событий.

## MVP

Цель MVP — дать одному человеку рабочий инструмент для ежедневного использования.

Минимальный путь:

```text
Telegram bot -> Memory Core -> Supabase/Postgres -> Search/Recall
```

Telegram — первый интерфейс.

Supabase — база данных PostgreSQL.

Railway — деплой.

OpenAI — позже, для транскрибации, анализа и умного поиска.
